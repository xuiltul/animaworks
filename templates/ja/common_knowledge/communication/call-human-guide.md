# call_human ガイド — 人間への通知と返信の受け取り

## 概要

`call_human` は人間の管理者に通知を送るツール。**一方通行ではない** — 人間が Slack のスレッドで返信すると、その返信は送信元 Anima の Inbox に自動配信される。

## 送信

```
call_human(
    subject="件名",
    body="本文（状況・試したこと・依頼内容を含める）",
    priority="normal"  # "normal" | "high" | "urgent"
)
```

| パラメータ | 必須 | 説明 |
|-----------|------|------|
| `subject` | MUST | 件名（簡潔に） |
| `body` | MUST | 本文（状況・試したこと・依頼を含める） |
| `priority` | MAY | `normal`（デフォルト）, `high`, `urgent` |

### いつ使うか

- 緊急度が「高」の問題（データ損失リスク、セキュリティ、サービス停止）
- 自分の判断範囲を超える決定が必要な場合
- トップレベル Anima で上司がいない場合のエスカレーション

詳細な判断基準は `troubleshooting/escalation-flowchart.md` を参照。

### 制約

- `call_human` は DM レート制限（30/h, 100/day）の **対象外**
- 緊急時でも制限を気にせず送信できる

## 返信の受け取り

### 仕組み

1. `call_human` で通知を送ると、Slack にメッセージが投稿される
2. 人間がそのメッセージの **スレッド（thread）** で返信する
3. 返信が送信元 Anima の Inbox に自動ルーティングされる
4. 次の Inbox 処理サイクル（通常2秒以内に検知）で返信が処理される

### 返信メッセージの特徴

受信した返信メッセージは以下の属性を持つ:

- `source`: `"slack"`（Slack 経由の返信）
- `from_person`: `"slack:U..."` 形式（Slack ユーザー ID）
- 通常の Inbox メッセージと同様に処理される

### 返信を待つ場合

`call_human` を送った後、返信を待つ必要がある場合:

1. 待機状態を `state/current_task.md` に記録する
2. 返信は次の Inbox 処理で自動的に届く
3. 即座に返信がなくても、人間が返信した時点で自動配信される

### 返信に応答する

返信を受け取ったら、通常の Inbox メッセージと同様に応答できる。ただし、返信先が Slack ユーザーのため `send_message` ではなく、チャット応答またはさらなる `call_human` で対応する。

## 注意事項

- 返信ルーティングは **Bot Token モード**（`chat.postMessage`）で送信された通知のみ対応。Webhook モードでは返信が届かない（管理者設定の問題であり、Anima 側で制御するものではない）
- 返信マッピングは **7日間** 保持される。7日以上経過したスレッドへの返信は届かない
- 人間からの返信がない場合でも、必要に応じて再度 `call_human` で催促できる
