# completion_gate — 完了前検証ガイド

最終回答の品質を保証するための自己検証メカニズム。
全実行モード（S/A/B/C/D/G）で利用可能。

## 概要

`completion_gate` は、Anima が最終回答を出す前に以下のチェックリストを実行させる仕組み:

- [ ] 元の指示を再読し、各要件が対処されているか確認
- [ ] 証拠が**このセッション内**に存在する（仮定ではない）
- [ ] 要求されたものが簡略化・省略されていない
- [ ] 独立したレビュアーが「完了」と判断できる品質

## モード別の動作

| モード | 実装方式 | 強制力 |
|--------|---------|--------|
| **S** (Agent SDK) | Stop hook が初回停止をブロックし、チェックリストを `reason` として注入。2回目の停止で通過 | 自動（ツール呼び出し不要） |
| **A** (LiteLLM) | `completion_gate` ツール + マーカーファイル。未呼び出し時に1回リトライを強制 | 自動（1回リトライ） |
| **B** (Assisted) | ツールとして提供。強制リトライなし | ガイダンスのみ |
| **C/D/G** (Codex/Cursor/Gemini) | MCP 経由でツール提供 | ガイダンスのみ |

## トリガー適用ルール

| トリガー | 適用 | 理由 |
|---------|------|------|
| `chat` | ✅ | 人間との対話品質を保証 |
| `task:*` | ✅ | タスク実行の完了品質を保証 |
| `cron:*` | ✅ | 定時タスクの品質を保証 |
| `heartbeat` | ❌ | Observe/Plan/Reflectのみ。最終回答を出さない |
| `inbox:*` | ❌ | 軽量返信のみ。検証オーバーヘッド不要 |
| `consolidation:*` | ❌ | ツールリスト自体から除外 |

## Mode A での使い方

Mode A（LiteLLM）では、最終回答の直前に `completion_gate` ツールを呼ぶことが期待される。呼び忘れた場合は1回だけリマインダーが表示される。

```
completion_gate()
```

戻り値としてチェックリストが返される。問題がなければそのまま回答を出力し、問題が見つかれば修正した回答を出力する。

## 注意事項

- チェックリストの検証プロセスを**ナレーションしない**こと。内部で確認し、問題がなければそのまま終了
- Mode S では `completion_gate` という名前のツール呼び出しは不要（Stop hook が自動処理）
- 信頼レベルは `trusted`（プロンプトインジェクション防御の対象外）
