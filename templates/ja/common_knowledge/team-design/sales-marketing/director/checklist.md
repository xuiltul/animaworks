# Sales & Marketing Director — 品質チェックリスト

machine 使用有無に関わらず、Director が成果物の品質を担保するためのチェックリスト。

---

## A. コンテンツ QC（Creator の draft-content.md 検証時）

- [ ] content-plan.md の目的・ターゲット・キーメッセージが反映されているか
- [ ] Brand Voice（トーン・禁止表現・用語統一）に準拠しているか
- [ ] ファネルステージ（TOFU/MOFU/BOFU）に適した CTA があるか
- [ ] ターゲットペルソナに適切な深度・表現になっているか
- [ ] 事実の誤記がないか（社名・数値・日付）

---

## B. コンプライアンス（二次判定）

- [ ] 誇大表現・優良誤認にあたる記述がないか
- [ ] 薬機法に抵触する可能性のある表現がないか（医療・健康系の場合）
- [ ] 景品表示法の有利誤認にあたる記述がないか
- [ ] 特定電子メール法のオプトイン要件を満たしているか（メール配信の場合）
- [ ] 個人情報の取り扱いに問題がないか
- [ ] {COMPLIANCE_REVIEWER} へのレビュー依頼が必要か判定したか

---

## C. 営業コンテンツ（machine 出力検証）

- [ ] 提案書のカスタマイズがターゲットに適しているか
- [ ] 差別化ポイントが正確で裏付けがあるか
- [ ] 競合情報が最新か（取得日を確認）
- [ ] バトルカード・反論対応の内容が事実に基づいているか
- [ ] ROI 計算の前提条件が明記されているか

---

## D. パイプライン管理

### Campaign Pipeline Tracker

- [ ] 全コンテンツ企画のステージが最新に更新されているか
- [ ] 停滞（2週間以上変化なし）の企画を特定したか
- [ ] 完了した企画の効果測定を計画したか
- [ ] silent drop ゼロを確認したか

### Deal Pipeline Tracker

- [ ] 全商談のステージが最新に更新されているか
- [ ] 停滞（2週間以上変化なし）の案件を特定し、アクションを決定したか
- [ ] Lost 案件の理由が記録されているか
- [ ] CS Handoff 完了の案件に cs-handoff.md が作成されているか
- [ ] silent drop ゼロを確認したか

---

## E. 委譲時の品質

### Creator への依頼前に

- [ ] content-plan.md の `status: approved` を確認したか
- [ ] ターゲット・キーメッセージ・構成指示が十分に明確か
- [ ] コンプライアンス注意事項が記載されているか（該当する場合）
- [ ] 期限（deadline）を設定したか

### SDR への指示前に

- [ ] アウトバウンドのターゲット条件が明確か
- [ ] メッセージのトーン・方針が指定されているか

### Researcher への依頼前に

- [ ] 調査目的と範囲が明確か
- [ ] 期限（deadline）を設定したか

---

## F. フィードバック対応

- [ ] Creator からの draft-content.md を全件確認したか
- [ ] SDR からの lead-report.md を Deal Pipeline Tracker に反映したか
- [ ] {COMPLIANCE_REVIEWER} からのコンプライアンスレビュー結果を反映したか
- [ ] Researcher からの research-report.md を活用したか
- [ ] Campaign Pipeline Tracker・Deal Pipeline Tracker の最終ステータスを更新したか
