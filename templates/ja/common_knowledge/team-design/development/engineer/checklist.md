# Engineer — 品質チェックリスト

machine 使用有無に関わらず、Engineer が成果物の品質を担保するためのチェックリスト。

---

## 実装詳細計画書（impl.plan.md）の品質

### plan.md との整合性

- [ ] plan.md のゴールが正確に反映されているか
- [ ] plan.md の完了条件が全て impl.plan.md のステップに対応しているか
- [ ] plan.md の制約条件に違反するステップがないか
- [ ] plan.md のスコープを逸脱する変更が含まれていないか

### 技術的妥当性

- [ ] 対象ファイルのパスが正確か（存在確認済みか）
- [ ] 変更内容がプロジェクトの技術スタックに適合しているか
- [ ] 依存関係と実装順序が正しいか
- [ ] 既存コードの構造・パターンと整合しているか

### 実行可能性

- [ ] 各ステップが具体的で machine に実行可能な粒度か
- [ ] ロールバック計画が記載されているか
- [ ] テストの作成・更新ステップが含まれているか

---

## 実装出力の品質

### コード品質

- [ ] プロジェクトのコーディング規約に準拠しているか
- [ ] 型ヒントが適切に付与されているか
- [ ] 適切なエラーハンドリングが実装されているか
- [ ] 不要なコード（デバッグ出力、コメントアウト）が残っていないか

### テスト

- [ ] 新規コードにユニットテストが追加されているか
- [ ] 既存テストが全件パスするか
- [ ] エッジケース（空値、境界値、大量データ）がテストされているか

### 設計適合性

- [ ] `git diff` の内容が impl.plan.md の変更内容と一致するか
- [ ] 意図しないファイルが変更されていないか
- [ ] plan.md の完了条件を全て満たしているか
- [ ] plan.md の制約条件に違反していないか

### 安全性

- [ ] 破壊的変更（既存 API の変更、DB スキーマ変更等）がある場合、plan.md で承認済みか
- [ ] セキュリティ上の問題（ハードコードされた秘密情報、入力バリデーション不足等）がないか
- [ ] パフォーマンスへの悪影響（N+1 クエリ、不要なループ等）がないか

---

## 引き継ぎ時の品質

### Reviewer / Tester への依頼前に

- [ ] 実装出力の検証（チェックポイント 2）を完了したか
- [ ] 変更内容の要約が伝わる形式になっているか
- [ ] `plan.md` へのリンクが含まれているか
- [ ] テスト実行コマンドが明記されているか
