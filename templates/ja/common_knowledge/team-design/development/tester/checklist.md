# Tester — 品質チェックリスト

machine 使用有無に関わらず、Tester がテスト品質を担保するためのチェックリスト。

---

## テスト計画書（test.plan.md）の品質

### テスト戦略

- [ ] テスト目的が plan.md のゴール・完了条件と対応しているか
- [ ] テスト種類の選択理由が明記されているか
- [ ] テスト対象の範囲が明確か（何をテストし、何をテストしないか）

### テスト観点の網羅性

- [ ] 機能要件のテストが含まれているか
- [ ] エラーハンドリングのテストが含まれているか
- [ ] 既存機能の回帰テストが考慮されているか
- [ ] 非機能要件（パフォーマンス、セキュリティ）が必要に応じて含まれているか

### 実行可能性

- [ ] テスト環境の前提条件が現実的か
- [ ] テストデータの準備方法が明記されているか
- [ ] テスト実行コマンドが正確か

---

## テストケース（test-cases.md）の品質

### カテゴリ網羅性

- [ ] 正常系: 主要なユーザーフローが全てカバーされているか
- [ ] 異常系: 不正入力、権限不足、リソース不足等の異常パターンが含まれているか
- [ ] エッジケース: 境界値、空値、最大値、特殊文字等が考慮されているか

### 個別テストケースの品質

- [ ] 前提条件が明確か（テストの独立性が保たれるか）
- [ ] 操作手順が再現可能な粒度で記述されているか
- [ ] 期待結果が具体的かつ検証可能か（「正しく動く」ではなく具体値）
- [ ] 優先度が適切に設定されているか

### plan.md との対応

- [ ] plan.md の完了条件が全てテストケースにマッピングされているか
- [ ] plan.md の制約条件の遵守がテストで確認されているか

---

## テスト結果（test-report.md）の品質

### 結果の完全性

- [ ] 全テストケースの結果が記録されているか（漏れがないか）
- [ ] 各結果にエビデンス（ログ、出力、スクリーンショット情報等）が含まれているか
- [ ] Skip されたテストの理由が記録されているか

### Fail 分析

- [ ] Fail の原因分類が正確か（実装 / テスト環境 / テストケースのいずれか）
- [ ] 偽陽性（False Positive）を識別して除外したか
- [ ] 偽陰性（False Negative）の可能性を検討したか
- [ ] 再現手順が具体的に記述されているか

### 不具合報告

- [ ] 再現手順が第三者が実行可能な形式か
- [ ] 期待結果と実際の結果が明確に対比されているか
- [ ] 重要度の判定が妥当か

---

## フィードバックの品質

### Engineer への報告

- [ ] Fail の具体的な再現手順が含まれているか
- [ ] 期待値と実際値が明確に記述されているか
- [ ] 修正後の再テスト方法が示されているか

### PdM への報告

- [ ] テスト全体のサマリー（Pass/Fail 数、合格判定）が含まれているか
- [ ] 重大な問題がある場合、影響範囲の評価が含まれているか
- [ ] 追加テストの必要性について所見が含まれているか
