# Strategy Director — machine 活用ガイド

Strategy Director が machine を使ってマーケットスキャン・定量的評価を実施するためのガイド。

> **重要**: `strategy-plan.md` は Director 自身が書く（machine に書かせない）。machine は「分析の実行」に使い、「判断」は Director が行う。

---

## 基本ルール

1. **計画書ファースト**: machine に渡す前に「何を分析させるか」を明確にする
2. **出力は draft**: machine の出力はそのまま最終成果物にしない。Director が検証する
3. **ファイルパス**: `state/plans/` に計画書を保存し、machine への instruction として渡す
4. **レート制限遵守**: machine の同時実行制限を守る
5. **機密データ注意**: machine には API キー・パスワードを渡さない

---

## Phase 1: 戦略設計（Director 自身が書く）

machine は使用しない。Director が以下のテンプレートに従い `strategy-plan.md` を作成する。

### strategy-plan.md テンプレート

```markdown
# 戦略計画書: {戦略名}

status: draft
author: {anima名}
date: {YYYY-MM-DD}
type: strategy-plan

## 戦略目的
{何を達成するか — 1〜3文}

## 対象市場
| 資産クラス | 銘柄/ペア | 時間軸 | 取引所/プラットフォーム |
|-----------|----------|--------|---------------------|
| {クラス} | {銘柄} | {時間軸} | {取引所} |

## リスクパラメータ
| パラメータ | 値 | 根拠 |
|-----------|-----|------|
| 最大ドローダウン | {%} | {根拠} |
| ポジション上限 | {金額 or ロット} | {根拠} |
| レバレッジ上限 | {倍率} | {根拠} |
| 損切りルール | {条件} | {根拠} |

## Performance Tracker からの引き継ぎ事項
| # | 前回の問題 | 前回ステータス | 今回の対応方針 |
|---|----------|-------------|-------------|
| S-{N} | {問題} | {ステータス} | {方針} |

## 戦略仮説
{エッジの仮説 — なぜ利益が出ると考えるか}

## 検証計画
1. {検証ステップ1}
2. {検証ステップ2}

## 完了条件
- [ ] バックテストで {条件} を満たすこと
- [ ] ペーパートレードで {期間} 検証すること
- [ ] Auditor の APPROVE を得ること

## 期限
{deadline}
```

## Phase 2: マーケットスキャン・定量分析（machine 活用）

Director が machine にマーケットデータの収集・定量分析を委託する。

### 用途と instruction テンプレート

#### 2-A: 市場環境スキャン

```
以下の市場について、直近{期間}のデータを収集・分析せよ:

対象: {銘柄/ペアリスト}
分析項目:
- 価格トレンド（方向性・ボラティリティ）
- 出来高変化
- 相関構造の変化
- 重要イベント（経済指標・規制変更等）

出力形式:
- 各銘柄の要約（3行以内）
- リスク要因のリスト
- 注目すべき異常値
```

#### 2-B: バックテスト結果の定量検証

```
以下のバックテスト結果を定量的に検証せよ:

対象: {backtest-report.md のパス}
検証項目:
- in-sample vs out-of-sample のパフォーマンス乖離
- パラメータ感度分析（±{N}% 変動時の影響）
- 最大ドローダウン期間とリカバリー期間
- 取引頻度と手数料影響

出力: 検証結果サマリー + 過学習リスク評価（高/中/低）
```

## Phase 3: PDCA 分析（machine 活用）

### 用途: Check フェーズのデータ収集

```
以下の戦略の直近{期間}パフォーマンスを分析せよ:

対象: {戦略名/ログファイルパス}
分析項目:
- P&L 推移（日次/週次）
- 勝率トレンド（改善/横ばい/悪化）
- ドローダウン推移
- エッジ decay の兆候（直近の勝率 vs 全期間の勝率）

出力: PDCA Check レポート（Act での判断材料として使用）
```

> **注意**: machine の出力は「Check」のデータ。「Act」の判断（続行/調整/停止）は Director 自身が行う。
