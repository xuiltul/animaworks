# Strategy Director — 品質チェックリスト

machine 使用有無に関わらず、Strategy Director が戦略品質と PDCA の回転を担保するためのチェックリスト。

---

## A. P&L モニタリング Gate

- [ ] 現在の累積 P&L を確認したか
- [ ] ドローダウンが閾値 `{max_drawdown_pct}` 以内であることを確認したか
- [ ] ドローダウンが閾値に接近（80%以上）している場合、対応策を検討したか
- [ ] 損益トレンド（改善/横ばい/悪化）を評価したか
- [ ] 全戦略の合算ポートフォリオリスクを確認したか

---

## B. 戦略品質 Gate

- [ ] バックテストの in-sample と out-of-sample でパフォーマンスに大きな乖離がないか
- [ ] パラメータ感度分析を実施し、パラメータの微小変動で結果が大きく変わらないか確認したか
- [ ] エッジの持続性を評価したか（直近の勝率 vs 全期間の勝率）
- [ ] スリッページ・手数料が現実的な値で反映されているか
- [ ] サバイバーシップバイアスが排除されているか
- [ ] ルックアヘッドバイアスが排除されているか

---

## C. リスク管理 Gate

- [ ] 全ポジションのサイズが `{position_limit}` 以内か
- [ ] レバレッジが `{leverage_limit}` 以内か
- [ ] 相関集中リスクを確認したか（同一方向の複数ポジション）
- [ ] 流動性リスクを確認したか（板の薄い銘柄でのポジション）
- [ ] 損切りルールが全戦略に設定されているか
- [ ] 最大同時保有ポジション数を確認したか

---

## D. PDCA 進捗 Gate

- [ ] 前回 Check の指摘事項が全件 Act に反映されたか
- [ ] performance-tracker が最新ステータスに更新されているか（silent drop ゼロ）
- [ ] ops-issue-tracker の未解消項目を確認したか
- [ ] PDCA の Act で下した判断（続行/調整/停止）に根拠が明記されているか
- [ ] 次回 Check の予定日が設定されているか

---

## E. 委譲時の品質

### Engineer / Analyst への依頼前に

- [ ] strategy-plan.md の `status: approved` を確認したか
- [ ] リスクパラメータが明記されているか
- [ ] 完了条件が具体的かつ検証可能か
- [ ] 期限（deadline）を設定したか
- [ ] Performance Tracker からの引き継ぎ事項が strategy-plan.md に含まれているか

### Auditor からのフィードバック対応

- [ ] Auditor の指摘事項を全件確認したか
- [ ] P&L 乖離の指摘に対して原因分析を実施したか
- [ ] 運用健全性の問題について対応策を決定したか
- [ ] performance-tracker / ops-issue-tracker を最終更新したか
