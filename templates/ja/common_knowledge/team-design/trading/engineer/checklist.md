# Trading Engineer — 品質チェックリスト

machine 使用有無に関わらず、Trading Engineer が実装とバックテストの品質を担保するためのチェックリスト。

---

## A. 実装品質

### strategy-plan.md との整合性

- [ ] strategy-plan.md のゴールが正確に反映されているか
- [ ] strategy-plan.md のリスクパラメータ（ドローダウン閾値、ポジション上限等）が実装に組み込まれているか
- [ ] strategy-plan.md の完了条件が全て impl.plan.md のステップに対応しているか
- [ ] strategy-plan.md のスコープを逸脱する変更が含まれていないか

### コード品質

- [ ] プロジェクトのコーディング規約に準拠しているか
- [ ] 適切なエラーハンドリングが実装されているか（API障害時のリトライ、ポジション保護）
- [ ] ランダムシードが固定されているか（再現可能性）
- [ ] ログ出力が十分か（エントリー/エグジット/損益/エラー）
- [ ] 不要なコード（デバッグ出力、コメントアウト）が残っていないか

### テスト

- [ ] 新規コードにユニットテストが追加されているか
- [ ] 既存テストが全件パスするか
- [ ] エッジケースがテストされているか（データ欠損時、API障害時、ゼロ除算等）

---

## B. バックテスト整合性

- [ ] ルックアヘッドバイアスが排除されているか（未来のデータを参照していないか）
- [ ] サバイバーシップバイアスが排除されているか（上場廃止銘柄等を含むデータか）
- [ ] スリッページが現実的な値で反映されているか
- [ ] 手数料が正確に反映されているか（取引所ごとの手数料率）
- [ ] 初期資金と取引単位が現実的か
- [ ] in-sample と out-of-sample の期間分割が適切か
- [ ] 十分な取引回数があるか（統計的に有意なサンプルサイズ）

---

## C. インフラ健全性

- [ ] bot プロセスが正常に稼働しているか確認したか
- [ ] API 接続（取引所・データソース）が正常か確認したか
- [ ] レイテンシが許容範囲内か
- [ ] ディスク容量・メモリ使用量に問題がないか
- [ ] ログローテーションが設定されているか
- [ ] dry-run 環境と本番環境が明確に分離されているか

---

## D. 引き継ぎ品質

### Auditor / Director への報告前に

- [ ] backtest-report.md が完備しているか（サマリー・条件・完了条件照合・再現手順）
- [ ] `status: reviewed` を付けたか（自分で検証済み）
- [ ] strategy-plan.md の完了条件との照合結果が記載されているか
- [ ] 再現手順（コマンド / スクリプト）が記載されているか
- [ ] 既知の制限事項が明記されているか
