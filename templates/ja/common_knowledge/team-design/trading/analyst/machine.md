# Market Analyst — machine 活用ガイド

Market Analyst が machine を使って大規模データ分析・モデル検証を実施するためのガイド。

> **重要**: machine は「データの収集・計算」に使う。「シグナルの解釈・確度の判断」は Analyst 自身が行う。

---

## 基本ルール

1. **計画書ファースト**: machine に渡す前に「何を分析させるか」を明確にする
2. **出力は draft**: machine の出力はそのまま market-analysis.md にしない。Analyst が検証する
3. **ファイルパス**: `state/plans/` に計画書を保存し、machine への instruction として渡す
4. **レート制限遵守**: machine の同時実行制限を守る
5. **機密データ注意**: machine には API キー・パスワードを渡さない

## Phase 1: データ収集・前処理（machine 活用）

### 用途と instruction テンプレート

#### 1-A: 価格データ収集・統計計算

```
以下の銘柄/ペアについて、直近{期間}のデータを収集・基本統計を計算せよ:

対象: {銘柄/ペアリスト}
データ項目:
- OHLCV（始値・高値・安値・終値・出来高）
- ボラティリティ（{期間}移動平均）
- リターン分布（正規性検定含む）
- 相関マトリクス（対象銘柄間）

出力形式: 統計サマリーテーブル + 異常値フラグ
```

#### 1-B: センチメント分析

```
以下のソースから{対象}に関するセンチメントを分析せよ:

ソース: {ニュースフィード / SNS / オンチェーンデータ}
対象: {銘柄/テーマ}
期間: 直近{期間}
分析項目:
- センチメントスコア推移
- キーワード頻度変化
- 異常なセンチメント変動の検出

出力: センチメントサマリー + 重要イベントリスト
```

## Phase 2: モデル検証（machine 活用）

#### 2-A: バックテスト vs ライブ乖離分析

```
以下の戦略のバックテスト結果とライブ結果を比較せよ:

バックテスト結果: {パス}
ライブ結果: {パス}
比較項目:
- 勝率の乖離
- P&L 分布の乖離
- スリッページ実績 vs 想定
- 約定率の乖離

出力: 乖離サマリー + 重要度判定（許容範囲内/要調査/要停止）
```

#### 2-B: ウォークフォワード分析

```
以下の戦略パラメータについてウォークフォワード分析を実施せよ:

戦略: {戦略名/コードパス}
パラメータ: {パラメータリスト}
学習期間: {期間}
検証期間: {期間}
ステップ: {ローリング期間}

出力: 各期間のパフォーマンス + オーバーフィット比率
```

## Phase 3: 分析レポート支援（machine 活用）

### market-analysis.md テンプレート（Analyst が最終判断を加えて完成）

```markdown
# 市場分析レポート: {分析対象}

status: draft
author: {anima名}
date: {YYYY-MM-DD}
type: market-analysis

## エグゼクティブサマリー
{1〜3文の結論}

## 分析対象と期間
| 銘柄/ペア | 時間軸 | 分析期間 |
|----------|--------|---------|
| {銘柄} | {時間軸} | {期間} |

## シグナル

| # | シグナル | 方向 | 確度 | 統計的根拠 | 前提条件 |
|---|---------|------|------|----------|---------|
| 1 | {シグナル} | {買/売/中立} | {高/中/参考} | {根拠} | {前提} |

## リスク要因
- {リスク1}
- {リスク2}

## 前提条件と限界
- {前提1: この分析が有効な条件}
- {限界1: この分析が対応できないケース}
```
