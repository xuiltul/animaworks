# Market Analyst — 品質チェックリスト

machine 使用有無に関わらず、Market Analyst がシグナルとデータの品質を担保するためのチェックリスト。

---

## A. シグナル品質

- [ ] シグナルに統計的根拠（p値、信頼区間、サンプルサイズ等）が付記されているか
- [ ] 確度が3段階（高確度/中確度/参考程度）で明示されているか
- [ ] 偽シグナル率（False Positive Rate）を評価したか
- [ ] 過去の類似シグナルの的中率を確認したか
- [ ] 複数の独立したデータソースで裏付けが取れているか
- [ ] 相場のレジーム（トレンド/レンジ/ボラティリティ）を考慮した分析か

---

## B. データ品質

- [ ] 使用したデータソースの信頼性を確認したか
- [ ] データの鮮度（最終更新日時）を確認したか
- [ ] 欠損値・異常値の処理方法を明記したか
- [ ] データの取得期間がシグナルの時間軸に対して十分か
- [ ] 複数のデータソース間で矛盾がないか確認したか

---

## C. モデル検証

- [ ] バックテスト vs ライブのパフォーマンス乖離を確認したか
- [ ] ウォークフォワード分析で過学習の兆候がないか検証したか
- [ ] レジーム変化（市場環境の構造的変化）に対するモデルの頑健性を評価したか
- [ ] パラメータ感度分析を実施したか（微小変動で結果が大きく変わらないか）
- [ ] サバイバーシップバイアスが排除されているか
- [ ] ルックアヘッドバイアスが排除されているか

---

## D. レポート品質

- [ ] market-analysis.md のフォーマットに従っているか
- [ ] エグゼクティブサマリーが結論を簡潔に伝えているか
- [ ] 全シグナルに確度と統計的根拠が付記されているか
- [ ] 前提条件と限界が明示されているか
- [ ] リスク要因が列挙されているか
- [ ] 「高確度」のシグナルに十分な裏付けがあるか（裏付けが弱い場合は確度を下げる）

---

## E. machine 出力の検証

### machine にデータ収集・計算を委託した場合

- [ ] 計算結果を独立した方法で検算したか（サンプリングチェック可）
- [ ] 異常値・外れ値が正しく処理されているか
- [ ] データの取得範囲・期間が指示通りか
- [ ] machine が生成した統計値の単位・スケールが正しいか
- [ ] 出力から Analyst 自身の判断を加えた上で market-analysis.md を完成させたか
