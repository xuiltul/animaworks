# Risk Auditor — 品質チェックリスト

machine 使用有無に関わらず、Risk Auditor が検証品質を担保するためのチェックリスト。
このチェックリストは、トレーディングチームで報告された3つの致命的問題を構造的に防止するために設計されている。

---

## A. P&L 検証 Gate（「お金が減っても気にしない」対策）

- [ ] P&L を Director/Engineer の報告とは独立に計算し、照合したか
- [ ] ドローダウンが閾値 `{max_drawdown_pct}` 以内であることを確認したか
- [ ] ドローダウンが閾値の 80% 以上の場合、Director に早期警告を出したか
- [ ] 勝率トレンド（改善/横ばい/悪化）を評価したか
- [ ] エッジの decay（直近の勝率 vs 全期間の勝率の乖離）を検出したか
- [ ] 「まだ大丈夫」「一時的な下落」「想定内」等の楽観的判断を検証したか
- [ ] Director が「継続」と判断した戦略について、停止した場合の最悪シナリオを検討したか
- [ ] バックテスト P&L とライブ P&L の乖離が `{pl_divergence_threshold}` 以内か確認したか

---

## B. 執行検証 Gate（「実トレードの検証をしていない」対策）

- [ ] bot プロセスが稼働していることを確認したか（PID・最終ログタイムスタンプ）
- [ ] API 接続（取引所・データソース）が正常であることを確認したか
- [ ] 注文が実際に約定しているか確認したか（空回りしていないか）
- [ ] dry-run（ペーパートレード）と本番が明確に区別されているか
- [ ] 約定拒否・タイムアウトが異常に多くないか
- [ ] スリッページ実績が想定範囲内か
- [ ] bot の停止時間が `{max_downtime}` を超えていないか

---

## C. 資産追跡 Gate（「ウォレットトラッキングを忘れる」対策）

- [ ] 全取引所/ウォレットの残高を照合したか
- [ ] 期待残高と実残高の差異が `{balance_tolerance}` 以内か
- [ ] 入出金の記録が正確か（入金/出金/送金のログ確認）
- [ ] ポジション棚卸しを実施したか（保有銘柄/数量/評価額）
- [ ] 前回照合時からの残高変動に説明がつくか（P&L + 入出金 = 残高変動）

---

## D. リスク準拠 Gate

- [ ] 全ポジションのサイズが `{position_limit}` 以内か
- [ ] レバレッジが `{leverage_limit}` 以内か
- [ ] 相関集中リスクを確認したか（同一方向の複数ポジション）
- [ ] 損切りルールが正しく機能しているか（損切りが発動すべき場面で発動したか）
- [ ] 最大同時保有ポジション数を確認したか

---

## E. Carry-forward Gate

- [ ] performance-tracker の全項目のステータスが更新されているか
- [ ] 前回「未解消」だった項目が全件言及されているか（silent drop ゼロ）
- [ ] ops-issue-tracker の全項目のステータスが更新されているか
- [ ] 新規検出した問題が tracker に登録されているか
- [ ] ステータスの遷移が論理的か（「解消」→「未対応」への逆行がないか）

---

## F. メタ検証の品質

### machine 検証結果の検証

- [ ] P&L 計算結果をサンプリングチェックしたか
- [ ] 残高照合の数値が正確か
- [ ] 誤検出（False Positive）を除去したか
- [ ] 見落とし（False Negative）を補完したか

### フィードバックの品質

- [ ] Critical 指摘に具体的な数値根拠と推奨アクションが含まれているか
- [ ] 楽観バイアスの指摘に最悪シナリオが添えられているか
- [ ] carry-forward 漏れの指摘に前回レポートの該当箇所が引用されているか
- [ ] performance-review.md と ops-health-report.md が完備しているか
