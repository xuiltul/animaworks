# Financial Auditor — 品質チェックリスト

machine 使用有無に関わらず、Financial Auditor が検証品質を担保するためのチェックリスト。

---

## A. Assumption Challenge Gate

- [ ] Director の仮定（「季節要因」「一時的」「業界平均」等）を全件リストアップしたか
- [ ] 「季節要因」の全判断を過去データ（少なくとも12ヶ月）で裏付け確認したか
- [ ] 「一時的」の全判断に追跡期間と再発基準が設定されているか
- [ ] 楽観的な予測に対して悲観シナリオ（感応度分析）を検討したか
- [ ] 「業界平均」「業界標準」の全主張に具体的なベンチマークデータがあるか
- [ ] 前回より低リスク評価に変更された項目に、十分な根拠があるか

---

## B. Data Lineage Verification

- [ ] analysis-report 内の全数値がソースデータまで遡れるか
- [ ] 推定値に「推定」マーカーが付いているか
- [ ] 孤立した計算（ソースに紐づかない数値）がないか
- [ ] Variance Tracker の全件が analysis-report に反映されているか（silent drop 検出）
- [ ] 前回 Critical → 今回言及なしの項目がないか（silent drop 検出）

---

## C. 数値正確性（Numerical Accuracy）

- [ ] 主要指標の独立再計算を実施したか
- [ ] 会計恒等式（BS 等式・TB 均衡）が成立しているか
- [ ] Director の数値検証（assert 文等）の結果を確認したか
- [ ] 代替計算手法での検算を実施したか
- [ ] 変動率の計算が正確か（分母・分子の取り違いがないか）
- [ ] 端数処理・丸めの一貫性を確認したか

---

## D. Cross-Entity Reconciliation（連結・グループ分析時のみ）

- [ ] 法人間残高が相殺消去されているか
- [ ] 消去仕訳が完全か（漏れがないか）
- [ ] 二重計上がないか
- [ ] 為替換算レートと換算方法が一貫しているか

---

## E. メタ検証の品質

### machine 検証結果の検証

- [ ] 誤検出（False Positive）を除去したか
- [ ] 見落とし（False Negative）を補完したか
- [ ] 重要度の判定（Critical/Warning/Info）が妥当か
- [ ] 指摘内容が事実に基づいているか（推測や思い込みでないか）

### フィードバックの品質

- [ ] Critical 指摘に具体的な反証データと推奨修正が含まれているか
- [ ] Assumption Challenge の指摘に過去データや悲観シナリオが添えられているか
- [ ] 建設的なトーンになっているか（問題の指摘 + 改善提案）
- [ ] silent drop の指摘に前回レポートの該当箇所が引用されているか
