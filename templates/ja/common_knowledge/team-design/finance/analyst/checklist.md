# Data Analyst — 品質チェックリスト

多段階検証プロセスに沿って、Data Analyst がデータ抽出品質を担保するためのチェックリスト。

---

## A. 抽出正確性（Extraction Accuracy）

- [ ] 全数値を原典と照合したか
- [ ] 第1段階（機械的抽出）と第2段階（構造確認）の両方を実施したか
- [ ] 合計値・小計が個別項目の合算と一致しているか
- [ ] 桁数・単位が正確か（千円/百万円/円の混在がないか）
- [ ] 推定値や概算値を使用していないか
- [ ] 不明項目を推測で埋めていないか（空欄としているか）

---

## B. 構造整合性（Structural Integrity）

- [ ] 勘定科目体系と整合しているか
- [ ] 必須項目（資産・負債・純資産・収益・費用の主要科目）が欠損していないか
- [ ] 期間（年月日）の指定が正確か
- [ ] 前期データとの構造的一貫性があるか（科目の追加・削除・名称変更を把握しているか）
- [ ] 複数ソースからのデータ間で矛盾がないか

---

## C. 出典記録（Source Documentation）

- [ ] 全数値に出典（原典ファイル名・ページ番号等）が記録されているか
- [ ] 原典データの取得日・バージョンが記録されているか
- [ ] 原典からの転記方法（手入力 / ツール抽出）が記録されているか

---

## D. 報告品質

- [ ] 抽出件数が記載されているか
- [ ] 検証結果（合格 / 差異あり）が明記されているか
- [ ] 差異やエラーが発見された場合、詳細が記載されているか
- [ ] 不明項目の一覧と理由が記載されているか
- [ ] Director が分析に使用できる形式に構造化されているか
