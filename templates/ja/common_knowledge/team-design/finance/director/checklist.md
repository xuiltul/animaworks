# Finance Director — 品質チェックリスト

machine 使用有無に関わらず、Finance Director が成果物の品質を担保するためのチェックリスト。

---

## A. 前回分析との整合性（Variance Tracker Gate）

- [ ] 前回分析の重要差異が全件リストアップされているか
- [ ] 各差異に対して「解消 / 継続監視 / 調査中 / 悪化」のステータスが付いているか
- [ ] 「継続監視」の項目に追跡期間と判断基準が併記されているか
- [ ] 「調査中」の項目に原因特定の進捗が記載されているか
- [ ] Variance Tracker のステータスが全件更新されているか（silent drop ゼロ）

---

## B. 財務諸表整合性（Financial Statement Integrity）

- [ ] BS 恒等式（資産 = 負債 + 純資産）が成立しているか
- [ ] 試算表の借方合計と貸方合計が一致しているか
- [ ] キャッシュフロー計算書の期末残高と BS 現金残高が整合しているか
- [ ] 小計・合計が個別項目の合算と一致しているか
- [ ] ソースデータからの転記に誤りがないか（Data Lineage 確認）
- [ ] 期間（年月日）の指定が正確か

---

## C. 数値検証（Numerical Verification）

- [ ] 全計算をプログラム的に検証したか（LLM の暗算に依存していないか）
- [ ] assert 文等で主要な恒等式・整合性を検証したか
- [ ] 全チェックが通過した後にのみ報告しているか
- [ ] 端数処理・丸めが一貫しているか
- [ ] 通貨換算レートの出典と日付が記録されているか
- [ ] 不一致があった場合、原因を特定するまでやり直したか

---

## D. 分析種別固有チェック

対象分析の種類に応じて、以下から該当する項目を確認する:

### D-1. 月次試算表分析

- [ ] 前期比較（前月・前年同月）が含まれているか
- [ ] 重要な変動（閾値超過）が全て検出・報告されているか
- [ ] 変動の原因分析（季節性・一時要因・構造変化）が記載されているか
- [ ] 異常値の判定に客観的な基準があるか

### D-2. ポートフォリオ評価

- [ ] 時価評価の基準日と評価方法が明記されているか
- [ ] ベンチマーク・参考指標との比較が含まれているか
- [ ] 評価損益の計算が正確か
- [ ] 資産クラス別の分析が含まれているか

### D-3. 連結・グループ分析

- [ ] 法人間取引（内部貸借・内部売上等）の相殺消去が正しく処理されているか
- [ ] 為替換算レートと換算方法が明記されているか
- [ ] 各法人の個別データと連結後のデータが整合しているか
- [ ] 二重計上がないか

### D-4. キャッシュフロー分析

- [ ] 資金繰り予測の前提条件が明記されているか
- [ ] 返済スケジュールとの整合性が確認されているか
- [ ] 資金ショートリスクの評価が含まれているか

---

## E. 出力品質

- [ ] 全数値にデータ系譜（ソースデータ・フィールド）があるか
- [ ] 推定値に「推定」マーカーが付いているか
- [ ] 非財務メンバーにも理解できる平易な説明になっているか
- [ ] 推奨アクションが全リスク項目に付いているか
- [ ] 事実の誤記がないか（法人名・期間・金額・単位）

---

## F. 委譲時の品質

### Auditor への依頼前に

- [ ] analysis-report.md の `status: reviewed` を確認したか
- [ ] analysis-plan.md（分析計画）が添付されているか
- [ ] Variance Tracker が最新ステータスに更新されているか
- [ ] 数値検証（assert 文等）の結果が含まれているか
- [ ] Auditor が検証に必要な情報（ソースデータ・前回レポート）がアクセス可能か
- [ ] 期限（deadline）を設定したか

### フィードバック対応

- [ ] Auditor からのフィードバックを全件確認したか
- [ ] 仮定への指摘に対して過去データで再検証を実施したか
- [ ] Data Lineage の指摘に対してソースを再確認したか
- [ ] Variance Tracker の最終ステータスを更新したか
