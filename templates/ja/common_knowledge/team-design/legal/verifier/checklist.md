# Legal Verifier — 品質チェックリスト

machine 使用有無に関わらず、Legal Verifier が検証品質を担保するためのチェックリスト。

---

## A. 楽観バイアス検出（Devil's Advocate Gate）

- [ ] 「受容可」「追加交渉不要」と判定された全項目を再評価したか
- [ ] 各「受容可」項目について「相手方がこの条項を最大限利用した場合の最悪シナリオ」を検討したか
- [ ] 「NTT側が拒否する理由がない」等の推測的表現がないか
- [ ] 「交渉の友好性を損なうリスク」等の非法的理由でリスク評価を下げていないか
- [ ] リスク評価が前回より下がった項目に、十分な法的根拠があるか
- [ ] 努力義務の水準（best efforts / reasonable efforts）が甘い方に解釈されていないか

---

## B. 見落とし検出（Coverage Verification）

- [ ] 前回監査の全指摘事項が今回レポートに反映されているか
- [ ] 前回 Critical → 今回言及なしの項目がないか（silent drop 検出）
- [ ] 契約書の条項数と、レポートで言及された条項数を照合したか
- [ ] 修正箇所以外の条項が変更されていないことを確認したか
- [ ] 定義条項の変更がリスク評価に反映されているか
- [ ] 付属書・別紙の確認漏れがないか

---

## C. 法的正確性（Legal Accuracy）

- [ ] 引用された法令（条文番号）が正確か
- [ ] 引用された判例（裁判所・年月日・事件番号）が正確か
- [ ] 「業界標準」の主張に裏付けがあるか（裏付けがない場合は Researcher に確認依頼済みか）
- [ ] 法的用語の使い方が正確か（「善管注意義務」「瑕疵担保」等の誤用がないか）
- [ ] 準拠法に基づいた分析がされているか（日本法 / 外国法の区別）

---

## D. 交渉戦略の妥当性

- [ ] 「署名前に確認すべき事項」と「署名後でよい事項」の区分が適切か
- [ ] 署名後の交渉力変動を考慮した評価になっているか
- [ ] 相手方の交渉ポジション分析が含まれているか
- [ ] 交渉優先度の順位が妥当か（リスクの大きさ × 交渉可能性）
- [ ] 推奨アクションが具体的かつ実行可能か

---

## E. メタ検証の品質

### machine 検証結果の検証

- [ ] 誤検出（False Positive）を除去したか
- [ ] 見落とし（False Negative）を補完したか
- [ ] 重要度の判定（Critical/Warning/Info）が妥当か
- [ ] 指摘内容が事実に基づいているか（推測や思い込みでないか）

### フィードバックの品質

- [ ] Critical 指摘に具体的な根拠と推奨修正が含まれているか
- [ ] 楽観バイアスの指摘に最悪シナリオが添えられているか
- [ ] 建設的なトーンになっているか（問題の指摘 + 改善提案）
- [ ] carry-forward 漏れの指摘に前回レポートの該当箇所が引用されているか
