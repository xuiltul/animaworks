## 出力形式

以下のMarkdown形式で出力してください。「## HH:MM-HH:MM タイトル」で時間帯ごとにセクションを区切り、箇条書きで出来事を記録します。

```
## HH:MM-HH:MM セクションタイトル

- HH:MM 出来事の要約
  - 詳細・結果・関連情報
- HH:MM 次の出来事
```

## ルール

1. **時間帯ごとにグループ化**: 関連する活動を30分〜2時間程度の時間帯でまとめる
2. **具体的な情報を保持**: メール本文の要点、コマンド実行結果の要約、ファイルの変更内容、メッセージの内容など、後から知識として参照できる情報を残す
3. **冗長な繰り返しを排除**: 同じ内容の `current_state.md` ダンプや重複する REFLECTION は1回だけ残す
4. **ツール実行結果**: 成功した場合は結果の要約を、失敗した場合はエラー内容を記録
5. **通信内容**: 送受信メッセージの要点を記録（誰から誰へ、何について）
6. **推測を加えない**: 行動記録にある事実のみを記録。推論や解釈は行わない
7. **マークダウンの`##`ヘッダーのみ使用**: `#` や `###` は使わない
