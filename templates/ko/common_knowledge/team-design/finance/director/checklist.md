# Finance Director — 품질 체크리스트

machine 사용 여부와 관계없이 Finance Director가 산출물 품질을 보장하기 위한 체크리스트.

---

## A. 전회 분석과의 일관성(Variance Tracker Gate)

- [ ] 전회 분석의 중요 차이가 전건 나열되어 있는가
- [ ] 각 차이에 대해 「해소 / 지속 모니터링 / 조사 중 / 악화」 상태가 붙어 있는가
- [ ] 「지속 모니터링」 항목에 추적 기간과 판단 기준이 병기되어 있는가
- [ ] 「조사 중」 항목에 원인 특정 진행이 기재되어 있는가
- [ ] Variance Tracker 상태가 전건 갱신되었는가(silent drop 제로)

---

## B. 재무제표 일관성(Financial Statement Integrity)

- [ ] BS 항등식(자산 = 부채 + 자본)이 성립하는가
- [ ] 시산표의 차변 합계와 대변 합계가 일치하는가
- [ ] 현금흐름표의 기말 잔액과 BS 현금 잔액이 일치하는가
- [ ] 소계·합계가 개별 항목의 합산과 일치하는가
- [ ] 소스 데이터로부터의 전기에 오류가 없는가(Data Lineage 확인)
- [ ] 기간(연월일) 지정이 정확한가

---

## C. 수치 검증(Numerical Verification)

- [ ] 전 계산을 프로그램적으로 검증했는가(LLM 암산에 의존하지 않았는가)
- [ ] assert 등으로 주요 항등식·일관성을 검증했는가
- [ ] 전 체크 통과 후에만 보고했는가
- [ ] 끝수 처리·반올림이 일관되는가
- [ ] 통화 환산 레이트의 출처와 날짜가 기록되어 있는가
- [ ] 불일치가 있을 때 원인을 특정할 때까지 재작업했는가

---

## D. 분석 종류별 고유 체크

대상 분석 유형에 따라 아래에서 해당 항목을 확인한다:

### D-1. 월간 시산표 분석

- [ ] 전기 비교(전월·전년 동월)가 포함되어 있는가
- [ ] 중요한 변동(임계값 초과)이 전부 검출·보고되었는가
- [ ] 변동 원인 분석(계절성·일시 요인·구조 변화)이 기재되어 있는가
- [ ] 이상값 판정에 객관적 기준이 있는가

### D-2. 포트폴리오 평가

- [ ] 시가 평가의 기준일과 평가 방법이 명시되어 있는가
- [ ] 벤치마크·참고 지표와의 비교가 포함되어 있는가
- [ ] 평가손익 계산이 정확한가
- [ ] 자산 클래스별 분석이 포함되어 있는가

### D-3. 연결·그룹 분석

- [ ] 법인 간 거래(내부 대여·내부 매출 등)의 상계 제거가 올바르게 처리되었는가
- [ ] 환산 레이트와 환산 방법이 명시되어 있는가
- [ ] 각 법인의 개별 데이터와 연결 후 데이터가 일치하는가
- [ ] 이중 계상이 없는가

### D-4. 현금흐름 분석

- [ ] 자금 조달 예측의 전제 조건이 명시되어 있는가
- [ ] 상환 스케줄과의 일관성이 확인되었는가
- [ ] 자금 부족 리스크 평가가 포함되어 있는가

---

## E. 출력 품질

- [ ] 모든 수치에 데이터 계통(소스 데이터·필드)이 있는가
- [ ] 추정값에 「추정」 마커가 붙어 있는가
- [ ] 비재무 구성원도 이해할 수 있는 쉬운 설명인가
- [ ] 권장 액션이 전 리스크 항목에 붙어 있는가
- [ ] 사실 오기가 없는가(법인명·기간·금액·단위)

---

## F. 위임 시 품질

### Auditor에 의뢰하기 전에

- [ ] analysis-report.md의 `status: reviewed`를 확인했는가
- [ ] analysis-plan.md(분석 계획)가 첨부되어 있는가
- [ ] Variance Tracker가 최신 상태로 갱신되어 있는가
- [ ] 수치 검증(assert 등) 결과가 포함되어 있는가
- [ ] Auditor가 검증에 필요한 정보(소스 데이터·전회 보고)에 접근 가능한가
- [ ] 기한(deadline)을 설정했는가

### 피드백 대응

- [ ] Auditor로부터의 피드백을 전건 확인했는가
- [ ] 가정에 대한 지적에 대해 과거 데이터로 재검증을 실시했는가
- [ ] Data Lineage 지적에 대해 소스를 재확인했는가
- [ ] Variance Tracker의 최종 상태를 갱신했는가
