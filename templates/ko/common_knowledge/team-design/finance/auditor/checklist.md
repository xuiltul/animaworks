# Financial Auditor — 품질 체크리스트

machine 사용 여부와 관계없이 Financial Auditor가 검증 품질을 보장하기 위한 체크리스트.

---

## A. Assumption Challenge Gate

- [ ] Director 가정(「계절 요인」「일시적」「업계 평균」 등)을 전건 나열했는가
- [ ] 「계절 요인」 전 판단을 과거 데이터(최소 12개월)로 뒷받침 확인했는가
- [ ] 「일시적」 전 판단에 추적 기간과 재발 기준이 설정되어 있는가
- [ ] 낙관적 예측에 대해 비관 시나리오(민감도 분석)를 검토했는가
- [ ] 「업계 평균」「업계 표준」 전 주장에 구체적인 벤치마크 데이터가 있는가
- [ ] 전회보다 저위험 평가로 바뀐 항목에 충분한 근거가 있는가

---

## B. Data Lineage Verification

- [ ] analysis-report 내 모든 수치가 소스 데이터까지 소급 가능한가
- [ ] 추정값에 「추정」 마커가 붙어 있는가
- [ ] 고립된 계산(소스에 묶이지 않은 수치)이 없는가
- [ ] Variance Tracker 전건이 analysis-report에 반영되어 있는가(silent drop 검출)
- [ ] 전회 Critical → 이번 언급 없음 항목이 없는가(silent drop 검출)

---

## C. 수치 정확성(Numerical Accuracy)

- [ ] 주요 지표의 독립 재계산을 실시했는가
- [ ] 회계 항등식(BS 등식·TB 균형)이 성립하는가
- [ ] Director 수치 검증(assert 등) 결과를 확인했는가
- [ ] 대체 계산 방식으로 검산을 실시했는가
- [ ] 변동률 계산이 정확한가(분모·분자 혼동 없음)
- [ ] 끝수 처리·반올림의 일관성을 확인했는가

---

## D. Cross-Entity Reconciliation(연결·그룹 분석 시에만)

- [ ] 법인 간 잔액이 상계 제거되었는가
- [ ] 제거 분개가 완전한가(누락 없음)
- [ ] 이중 계상이 없는가
- [ ] 환산 레이트와 환산 방법이 일관되는가

---

## E. 메타 검증 품질

### machine 검증 결과의 검증

- [ ] 오검출(False Positive)을 제거했는가
- [ ] 놓침(False Negative)을 보완했는가
- [ ] 중요도 판정(Critical/Warning/Info)이 타당한가
- [ ] 지적 내용이 사실에 기반하는가(추측이나 착각이 아닌가)

### 피드백 품질

- [ ] Critical 지적에 구체적 반증 데이터와 권장 수정이 포함되어 있는가
- [ ] Assumption Challenge 지적에 과거 데이터나 비관 시나리오가 첨부되어 있는가
- [ ] 건설적 톤인가(문제 지적 + 개선 제안)
- [ ] silent drop 지적에 전회 보고의 해당 부분이 인용되어 있는가
