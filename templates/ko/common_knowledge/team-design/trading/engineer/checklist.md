# Trading Engineer — 품질 체크리스트

machine 사용 여부와 관계없이 Trading Engineer가 구현과 백테스트 품질을 보장하기 위한 체크리스트.

---

## A. 구현 품질

### strategy-plan.md와의 정합성

- [ ] strategy-plan.md 목표가 정확히 반영되어 있는가
- [ ] strategy-plan.md 리스크 파라미터(드로다운 임계, 포지션 상한 등)가 구현에 반영되어 있는가
- [ ] strategy-plan.md 완료 조건이 전건 impl.plan.md 스텝에 대응하는가
- [ ] strategy-plan.md 범위를 벗어난 변경이 포함되어 있지 않은가

### 코드 품질

- [ ] 프로젝트 코딩 규약을 준수하는가
- [ ] 적절한 에러 처리가 구현되어 있는가(API 장애 시 재시도, 포지션 보호)
- [ ] 랜덤 시드가 고정되어 있는가(재현 가능성)
- [ ] 로그 출력이 충분한가(진입/청산/손익/에러)
- [ ] 불필요한 코드(디버그 출력, 주석 처리)가 남아 있지 않은가

### 테스트

- [ ] 신규 코드에 유닛 테스트가 추가되어 있는가
- [ ] 기존 테스트가 전건 통과하는가
- [ ] 엣지 케이스가 테스트되어 있는가(데이터 결손 시, API 장애 시, 0으로 나누기 등)

---

## B. 백테스트 정합성

- [ ] 룩어헤드 바이어스가 제거되어 있는가(미래 데이터를 참조하지 않는가)
- [ ] 서바이버십 바이어스가 제거되어 있는가(상장 폐지 종목 등을 포함한 데이터인가)
- [ ] 슬리피지가 현실적인 값으로 반영되어 있는가
- [ ] 수수료가 정확히 반영되어 있는가(거래소별 수수료율)
- [ ] 초기 자금과 거래 단위가 현실적인가
- [ ] in-sample과 out-of-sample 기간 분할이 적절한가
- [ ] 충분한 거래 횟수가 있는가(통계적으로 유의한 표본 크기)

---

## C. 인프라 건전성

- [ ] bot 프로세스가 정상 가동 중임을 확인했는가
- [ ] API 연결(거래소·데이터 소스)이 정상인지 확인했는가
- [ ] 레이턴시가 허용 범위 내인가
- [ ] 디스크 용량·메모리 사용에 문제가 없는가
- [ ] 로그 로테이션이 설정되어 있는가
- [ ] dry-run 환경과 본환경이 명확히 분리되어 있는가

---

## D. 인계 품질

### Auditor / Director에 보고하기 전에

- [ ] backtest-report.md가 완비되어 있는가(요약·조건·완료 조건 대조·재현 절차)
- [ ] `status: reviewed`를 붙였는가(본인 검증 완료)
- [ ] strategy-plan.md 완료 조건과의 대조 결과가 기재되어 있는가
- [ ] 재현 절차(명령 / 스크립트)가 기재되어 있는가
- [ ] 알려진 제한 사항이 명시되어 있는가
