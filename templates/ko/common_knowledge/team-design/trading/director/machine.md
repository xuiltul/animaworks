# Strategy Director — machine 활용 가이드

Strategy Director가 machine으로 마켓 스캔·정량 평가를 수행하기 위한 가이드.

> **중요**: `strategy-plan.md`는 Director 본인이 작성한다(machine에 쓰게 하지 않는다). machine은 「분석 실행」에 쓰고 「판단」은 Director가 한다.

---

## 기본 규칙

1. **계획서 우선**: machine에 넘기기 전에 「무엇을 분석할지」를 명확히 한다
2. **출력은 draft**: machine 출력을 그대로 최종 결과물로 하지 않는다. Director가 검증한다
3. **파일 경로**: `state/plans/`에 계획서를 저장하고 machine에 넘기는 instruction으로 쓴다
4. **레이트 제한 준수**: machine 동시 실행 제한을 지킨다
5. **기밀 데이터 주의**: machine에 API 키·비밀번호를 넘기지 않는다

---

## Phase 1: 전략 설계(Director 본인이 작성)

machine은 사용하지 않는다. Director가 아래 템플릿에 따라 `strategy-plan.md`를 작성한다.

### strategy-plan.md 템플릿

```markdown
# 전략 계획서: {전략명}

status: draft
author: {anima명}
date: {YYYY-MM-DD}
type: strategy-plan

## 전략 목적
{무엇을 달성할지 — 1〜3문}

## 대상 시장
| 자산 클래스 | 종목/페어 | 시간축 | 거래소/플랫폼 |
|-----------|----------|--------|---------------------|
| {클래스} | {종목} | {시간축} | {거래소} |

## 리스크 파라미터
| 파라미터 | 값 | 근거 |
|-----------|-----|------|
| 최대 드로다운 | {%} | {근거} |
| 포지션 상한 | {금액 또는 랏} | {근거} |
| 레버리지 상한 | {배율} | {근거} |
| 손절 규칙 | {조건} | {근거} |

## Performance Tracker에서의 인계 사항
| # | 전회 문제 | 전회 상태 | 이번 대응 방침 |
|---|----------|-------------|-------------|
| S-{N} | {문제} | {상태} | {방침} |

## 전략 가설
{엣지 가설 — 왜 이익이 난다고 보는지}

## 검증 계획
1. {검증 단계1}
2. {검증 단계2}

## 완료 조건
- [ ] 백테스트에서 {조건}을 만족할 것
- [ ] 페이퍼 트레이드로 {기간} 검증할 것
- [ ] Auditor의 APPROVE를 받을 것

## 기한
{deadline}
```

## Phase 2: 마켓 스캔·정량 분석(machine 활용)

Director가 machine에 마켓 데이터 수집·정량 분석을 위임한다.

### 용도와 instruction 템플릿

#### 2-A: 시장 환경 스캔

```
다음 시장에 대해 최근{기간} 데이터를 수집·분석하라:

대상: {종목/페어 목록}
분석 항목:
- 가격 트렌드(방향성·변동성)
- 거래량 변화
- 상관 구조 변화
- 중요 이벤트(경제 지표·규제 변경 등)

출력 형식:
- 각 종목 요약(3줄 이내)
- 리스크 요인 목록
- 주목할 이상치
```

#### 2-B: 백테스트 결과 정량 검증

```
다음 백테스트 결과를 정량적으로 검증하라:

대상: {backtest-report.md 경로}
검증 항목:
- in-sample vs out-of-sample 성과 괴리
- 파라미터 민감도 분석(±{N}% 변동 시 영향)
- 최대 드로다운 기간과 회복 기간
- 거래 빈도와 수수료 영향

출력: 검증 결과 요약 + 과적합 리스크 평가(높음/중간/낮음)
```

## Phase 3: PDCA 분석(machine 활용)

### 용도: Check 단계 데이터 수집

```
다음 전략의 최근{기간} 성과를 분석하라:

대상: {전략명/로그 파일 경로}
분석 항목:
- P&L 추이(일/주)
- 승률 트렌드(개선/횡보/악화)
- 드로다운 추이
- 엣지 decay 징후(최근 승률 vs 전체 기간 승률)

출력: PDCA Check 리포트(Act 판단 자료로 사용)
```

> **주의**: machine 출력은 「Check」 데이터다. 「Act」 판단(계속/조정/정지)은 Director 본인이 내린다.
