# Risk Auditor — 품질 체크리스트

machine 사용 여부와 관계없이 Risk Auditor가 검증 품질을 보장하기 위한 체크리스트.
이 체크리스트는 트레이딩 팀에서 보고된 3가지 치명적 문제를 구조적으로 방지하기 위해 설계되었다.

---

## A. P&L 검증 Gate(「돈이 줄어도 신경 쓰지 않음」 대책)

- [ ] P&L을 Director/Engineer 보고와 독립적으로 계산·대조했는가
- [ ] 드로다운이 임계 `{max_drawdown_pct}` 이내임을 확인했는가
- [ ] 드로다운이 임계의 80% 이상일 때 Director에 조기 경고를 냈는가
- [ ] 승률 트렌드(개선/횡보/악화)를 평가했는가
- [ ] 엣지 decay(최근 승률 vs 전체 기간 승률 괴리)를 검출했는가
- [ ] 「아직 괜찮다」「일시적 하락」「예상 내」 등 낙관적 판단을 검증했는가
- [ ] Director가 「계속」이라고 판단한 전략에 대해 정지했을 때의 최악 시나리오를 검토했는가
- [ ] 백테스트 P&L과 라이브 P&L 괴리가 `{pl_divergence_threshold}` 이내인지 확인했는가

---

## B. 집행 검증 Gate(「실제 트레이드 검증을 하지 않음」 대책)

- [ ] bot 프로세스가 가동 중임을 확인했는가(PID·최종 로그 타임스탬프)
- [ ] API 연결(거래소·데이터 소스)이 정상임을 확인했는가
- [ ] 주문이 실제로 체결되었는지 확인했는가(공회전하지 않는지)
- [ ] dry-run(페이퍼 트레이드)과 본운용이 명확히 구분되는가
- [ ] 체결 거부·타임아웃이 비정상적으로 많지 않은가
- [ ] 슬리피지 실적이 가정 범위 내인가
- [ ] bot 정지 시간이 `{max_downtime}`를 초과하지 않았는가

---

## C. 자산 추적 Gate(「지갑 트래킹을 잊음」 대책)

- [ ] 전 거래소/지갑 잔고를 대조했는가
- [ ] 기대 잔고와 실 잔고 차이가 `{balance_tolerance}` 이내인가
- [ ] 입출금 기록이 정확한가(입금/출금/송금 로그 확인)
- [ ] 포지션 실사를 수행했는가(보유 종목/수량/평가액)
- [ ] 전회 대조 시부터 잔고 변동에 설명이 붙는가(P&L + 입출금 = 잔고 변동)

---

## D. 리스크 준수 Gate

- [ ] 전 포지션 크기가 `{position_limit}` 이내인가
- [ ] 레버리지가 `{leverage_limit}` 이내인가
- [ ] 상관 집중 리스크를 확인했는가(동일 방향 복수 포지션)
- [ ] 손절 규칙이 올바르게 동작하는가(손절이 나와야 할 때 나왔는지)
- [ ] 최대 동시 보유 포지션 수를 확인했는가

---

## E. Carry-forward Gate

- [ ] performance-tracker 전 항목 상태가 갱신되어 있는가
- [ ] 전회 「미해소」였던 항목이 전건 언급되어 있는가(silent drop 0)
- [ ] ops-issue-tracker 전 항목 상태가 갱신되어 있는가
- [ ] 신규 검출 문제가 tracker에 등록되어 있는가
- [ ] 상태 전이가 논리적인가(「해소」→「미대응」으로의 역행이 없는가)

---

## F. 메타 검증 품질

### machine 검증 결과의 검증

- [ ] P&L 계산 결과를 샘플링 체크했는가
- [ ] 잔고 대조 수치가 정확한가
- [ ] 오탐(False Positive)을 제거했는가
- [ ] 미탐(False Negative)을 보완했는가

### 피드백 품질

- [ ] Critical 지적에 구체적 수치 근거와 권장 액션이 포함되어 있는가
- [ ] 낙관적 편향 지적에 최악 시나리오가 붙어 있는가
- [ ] carry-forward 누락 지적에 전회 리포트 해당 구절이 인용되어 있는가
- [ ] performance-review.md와 ops-health-report.md가 완비되어 있는가
