# Market Analyst — machine 활용 가이드

Market Analyst가 machine으로 대규모 데이터 분석·모델 검증을 수행하기 위한 가이드.

> **중요**: machine은 「데이터 수집·계산」에 쓴다. 「시그널 해석·확도 판단」은 Analyst 본인이 한다.

---

## 기본 규칙

1. **계획서 우선**: machine에 넘기기 전에 「무엇을 분석할지」를 명확히 한다
2. **출력은 draft**: machine 출력을 그대로 market-analysis.md로 하지 않는다. Analyst가 검증한다
3. **파일 경로**: `state/plans/`에 계획서를 저장하고 machine에 넘기는 instruction으로 쓴다
4. **레이트 제한 준수**: machine 동시 실행 제한을 지킨다
5. **기밀 데이터 주의**: machine에 API 키·비밀번호를 넘기지 않는다

---

## Phase 1: 데이터 수집·전처리(machine 활용)

### 용도와 instruction 템플릿

#### 1-A: 가격 데이터 수집·통계 계산

```
다음 종목/페어에 대해 최근{기간} 데이터를 수집·기본 통계를 계산하라:

대상: {종목/페어 목록}
데이터 항목:
- OHLCV(시가·고가·저가·종가·거래량)
- 변동성({기간} 이동평균)
- 수익률 분포(정규성 검정 포함)
- 상관 행렬(대상 종목 간)

출력 형식: 통계 요약 테이블 + 이상치 플래그
```

#### 1-B: 센티먼트 분석

```
다음 소스에서 {대상}에 관한 센티먼트를 분석하라:

소스: {뉴스 피드 / SNS / 온체인 데이터}
대상: {종목/테마}
기간: 최근{기간}
분석 항목:
- 센티먼트 스코어 추이
- 키워드 빈도 변화
- 비정상 센티먼트 변동 검출

출력: 센티먼트 요약 + 중요 이벤트 목록
```

## Phase 2: 모델 검증(machine 활용)

#### 2-A: 백테스트 vs 라이브 괴리 분석

```
다음 전략의 백테스트 결과와 라이브 결과를 비교하라:

백테스트 결과: {경로}
라이브 결과: {경로}
비교 항목:
- 승률 괴리
- P&L 분포 괴리
- 슬리피지 실적 vs 가정
- 체결률 괴리

출력: 괴리 요약 + 중요도 판정(허용 범위 내/조사 필요/정지 필요)
```

#### 2-B: 워크포워드 분석

```
다음 전략 파라미터에 대해 워크포워드 분석을 수행하라:

전략: {전략명/코드 경로}
파라미터: {파라미터 목록}
학습 기간: {기간}
검증 기간: {기간}
스텝: {롤링 기간}

출력: 각 기간 성과 + 과적합 비율
```

## Phase 3: 분석 리포트 지원(machine 활용)

### market-analysis.md 템플릿(Analyst가 최종 판단을 더해 완성)

```markdown
# 시장 분석 리포트: {분석 대상}

status: draft
author: {anima명}
date: {YYYY-MM-DD}
type: market-analysis

## 실행 요약
{1〜3문 결론}

## 분석 대상과 기간
| 종목/페어 | 시간축 | 분석 기간 |
|----------|--------|---------|
| {종목} | {시간축} | {기간} |

## 시그널

| # | 시그널 | 방향 | 확도 | 통계적 근거 | 전제 조건 |
|---|---------|------|------|----------|---------|
| 1 | {시그널} | {매수/매도/중립} | {높음/중/참고} | {근거} | {전제} |

## 리스크 요인
- {리스크1}
- {리스크2}

## 전제 조건과 한계
- {전제1: 이 분석이 유효한 조건}
- {한계1: 이 분석이 대응하지 못하는 경우}
```
