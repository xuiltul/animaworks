## 未読メッセージ

以下の未読メッセージがあります。**原則として全てのメッセージに対応してください。**

{summary}

### 対応手順（原則）

1. 各メッセージを注意深く読む
2. 送信者に関する記憶を検索する（shared/users/, knowledge/ 等）
3. メッセージの内容に応じて適切な行動を取る
   - 質問 → 回答を調査・検索して返信
   - 依頼・タスク → タスクを実行し、結果を返信
   - 報告 → 内容を確認し、必要に応じて返信
   - 設定変更依頼 → 変更を実施し、完了を返信
4. **必ず send_message またはBashのsendコマンドで送信元に返信する**
   - 返信には元メッセージの id と thread_id を使ってスレッドを維持すること
   - 処理に時間がかかる場合でも、まず「承知しました、対応します」と返信すること

### 返信不要の例外（以下に該当する場合はスキップ可）

- **挨拶・お礼のみ**: 「おはようございます」「ありがとうございます」「了解です」等、情報伝達を含まない定型メッセージ
- **重複メッセージ**: 同一送信者から同一内容のメッセージが複数届いている場合、1通にのみ返信すれば可
- **自分が送ったメッセージへの確認応答**: 自分の報告に対する「了解」「確認しました」等
- **直近のハートビートで既に対応済み**: 前回のハートビートで同じ送信者に既に返信した場合

例外に該当する場合、エピソード記録に「メッセージ確認済み・返信不要」と記載すること。

### 禁止事項

- 重要な質問・依頼を読んだだけで返信せずに終わること
- 内部で「確認した」と記録するだけで、対応が必要なメッセージに通知しないこと
- メッセージの存在を無視すること
