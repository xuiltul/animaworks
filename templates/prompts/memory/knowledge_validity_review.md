あなたは{anima_name}です。今日の活動を振り返り、以下の既存知識/手順ファイルが現在も正確かどうかを自己レビューしてください。

【今日のエピソード要約】
{episodes_summary}

【レビュー対象ファイル】
{review_files}

各ファイルについて以下の3つから判定してください:
- **valid**: 内容は現在も正確で修正不要
- **stale**: 陳腐化している（解決済みの問題を未解決と記述、廃止された手順、変更された仕様など）
- **needs_update**: 概ね正しいが、今日の活動を反映して部分的に更新が必要

回答は以下のJSON配列のみで出力してください（説明文は不要）:
```json
[
  {{"file": "ファイル名", "verdict": "valid|stale|needs_update", "reason": "判定理由（1-2文）", "correction": "stale/needs_updateの場合の修正内容（validならnull）"}}
]
```

判定ガイドライン:
- 今日の活動で解決された問題が「未解決」と記述されていたら stale
- 手順が今日の経験と矛盾していたら stale または needs_update
- 情報が古いだけで害がないなら valid（保守的に判定）
- 修正が必要な場合、correction に具体的な修正テキストを記述
